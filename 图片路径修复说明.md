# 🖼️ 子目录图片不显示问题修复说明

## 问题描述

只有 `assets/images/` 根目录下的图片能正常显示，子目录（如 `home/`、`Moments/`、`carousel/`）的图片无法显示。

## 已应用的修复

### 1. 移除了自定义的图片路径处理

在 `nuxt.config.ts` 中，移除了 `assetFileNames` 中对图片的特殊处理，让 Vite 使用默认行为。Vite 在处理 `import.meta.glob` 导入的图片时，会自动处理路径映射关系。

**修改前：**
```typescript
assetFileNames: (assetInfo) => {
  if (assetInfo.name && /\.(png|jpe?g|gif|svg|webp|ico)$/i.test(assetInfo.name)) {
    return 'assets/images/[name]-[hash][extname]' // 这会导致所有图片都在根目录
  }
  return 'assets/[ext]/[name]-[hash].[ext]'
}
```

**修改后：**
```typescript
assetFileNames: 'assets/[ext]/[name]-[hash].[ext]'
```

这样 Vite 会使用默认行为，保留更多的路径信息。

## 如果问题依然存在

### 方案 1：检查构建后的文件结构

在 Cloudflare Dashboard 的构建日志中，或者在本地运行 `npm run build` 后，检查 `dist/_nuxt/assets/` 目录结构：
- 是否包含 `images/home/` 子目录？
- 是否包含 `images/Moments/` 子目录？
- 是否包含 `images/carousel/` 子目录？

### 方案 2：使用 public 目录（备选方案）

如果 Vite 的默认行为仍然有问题，可以考虑将图片移动到 `public/images/` 目录：

1. 将 `assets/images/` 下的图片复制到 `public/images/`
2. 保持相同的目录结构：`public/images/home/`、`public/images/Moments/` 等
3. 在代码中使用绝对路径引用：`/images/home/avatar.webp`

但这种方法需要修改代码中的图片引用方式。

### 方案 3：检查 import.meta.glob 的使用

确保 `import.meta.glob` 的使用方式正确：

```javascript
// 正确的方式
const imageModules = import.meta.glob('~/assets/images/carousel/*', {
  eager: true,
  import: 'default'
})
```
