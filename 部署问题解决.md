# 🔧 Cloudflare Pages 部署问题解决

## ❌ 错误：npm ci 失败

如果你遇到以下错误：
```
npm error Clean install a project
npm error
npm error Usage:
npm error npm ci
```

### 解决方案

#### 方法 1：修改 Cloudflare Pages 构建设置（推荐，适用于 GitHub 自动构建）

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入你的 **Workers & Pages** → **Pages** → 你的项目
3. 点击 **Settings** → **Builds & deployments**
4. 点击 **"Configure build"** 或 **"Edit configuration"**
5. 修改以下设置：
   - **Framework preset**: 选择 **None**（重要：不要选择 Nuxt）
   - **Build command**: `npm install && npm run build`（使用 npm install 代替 npm ci）
   - **Build output directory**: `dist`
   - **Root directory**: `/` (留空)
6. 在 **Environment variables** 中添加：
   - **变量名**: `NODE_VERSION`
   - **变量值**: `20`
   - **环境**: Production
7. 保存设置，重新触发部署

#### 方法 2：使用 npm install 代替 npm ci

在 Cloudflare Dashboard 中：
1. 进入项目设置
2. 找到 **Builds & deployments**
3. 在 **Environment variables** 中添加：
   - 变量名：`NPM_FLAGS`
   - 变量值：`--legacy-peer-deps`（如果需要）

或者直接修改构建命令为：
```
npm install && npm run build
```

#### 方法 3：确保 package-lock.json 存在

确保你的项目根目录有 `package-lock.json` 文件：

```bash
# 如果不存在，运行：
npm install

# 这会生成 package-lock.json
```

然后提交到 Git 仓库。

#### 方法 4：使用 Wrangler CLI 部署（绕过 Cloudflare Pages 构建）

如果 Cloudflare Pages 的自动构建一直有问题，可以使用 Wrangler CLI 直接部署：

```bash
# 1. 本地构建
npm install
npm run build

# 2. 使用 Wrangler 部署
npx wrangler pages deploy dist
```

这种方式会跳过 Cloudflare Pages 的自动构建，直接上传已构建的文件。

## 📋 Cloudflare Pages 正确配置

### 构建设置

在 Cloudflare Dashboard 中，确保以下设置：

| 设置项 | 值 |
|--------|-----|
| **Framework preset** | None（或 Nuxt） |
| **Build command** | `npm run build` |
| **Build output directory** | `dist` |
| **Root directory** | `/` |
| **Node.js version** | `20`（在环境变量中设置 `NODE_VERSION=20`） |

### 环境变量

在 **Settings** → **Environment variables** 中添加：

| 变量名 | 值 | 环境 |
|--------|-----|------|
| `ADMIN_PASSWORD` | 你的密码 | Production |
| `NODE_VERSION` | `20` | Production（可选） |

## 🚀 推荐部署流程

### 方式 A：Git 自动部署（推荐）

1. **确保 package-lock.json 存在**
   ```bash
   npm install
   git add package-lock.json
   git commit -m "Add package-lock.json"
   git push
   ```

2. **在 Cloudflare Dashboard 配置**
   - 连接 Git 仓库
   - 设置构建命令：`npm run build`
   - 设置输出目录：`dist`
   - 添加环境变量 `ADMIN_PASSWORD`

3. **推送代码触发部署**
   ```bash
   git push
   ```

### 方式 B：Wrangler CLI 手动部署（最稳定）

```bash
# 1. 本地安装依赖
npm install

# 2. 本地构建
npm run build

# 3. 部署到 Cloudflare
npx wrangler pages deploy dist --project-name=你的项目名
```

这种方式最稳定，因为：
- ✅ 不依赖 Cloudflare 的构建环境
- ✅ 可以本地测试构建是否成功
- ✅ 避免 npm ci 的问题

## 🔍 检查清单

部署前确保：

- [ ] ✅ `package-lock.json` 文件存在
- [ ] ✅ 本地可以成功运行 `npm run build`
- [ ] ✅ `dist` 文件夹生成成功
- [ ] ✅ `wrangler.jsonc` 中 KV 命名空间 ID 已配置
- [ ] ✅ Cloudflare Dashboard 中设置了 `ADMIN_PASSWORD` 环境变量
- [ ] ✅ Node.js 版本设置为 20（在环境变量中）

## 💡 如果还是失败

1. **查看构建日志**：在 Cloudflare Dashboard 中查看详细的构建日志
2. **尝试使用 npm install**：在环境变量中设置，或修改构建命令
3. **使用 Wrangler CLI**：这是最可靠的方式
4. **检查 Node.js 版本**：确保使用 Node.js 20

如果问题依然存在，请提供完整的构建日志，我可以帮你进一步诊断。
