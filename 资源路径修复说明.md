# 🔧 资源路径 404 错误修复说明

## 问题描述

所有静态资源（JS、CSS、图片）请求 `/_nuxt/` 路径时返回 404 错误：
- `/_nuxt/assets/js/entry-TFFHcxLw.js` → 404
- `/_nuxt/assets/css/entry-DMaWw1KN.css` → 404
- `/_nuxt/assets/images/Logo2-ZPKkck9k.webp` → 404

## 已应用的修复

### 1. 更新了 `nuxt.config.ts`

添加了以下配置：

```typescript
app: {
  buildAssetsDir: '/_nuxt/', // 确保资源路径正确
  // ...
}

nitro: {
  preset: 'cloudflare-pages',
  publicAssets: [
    {
      baseURL: '/',
      dir: 'public',
      maxAge: 60 * 60 * 24 * 7
    }
  ],
  prerender: {
    crawlLinks: false
  }
}
```

## 验证步骤

1. **提交并推送代码**
   ```bash
   git add nuxt.config.ts
   git commit -m "修复 Cloudflare Pages 资源路径问题"
   git push
   ```

2. **等待 Cloudflare Pages 自动构建**
   - 构建完成后，检查构建日志
   - 确认构建成功，没有错误

3. **测试网站**
   - 访问 `https://lcjnotes.pages.dev`
   - 打开浏览器开发者工具（F12）
   - 查看 Network 标签
   - 刷新页面，检查资源是否正常加载

## 如果问题依然存在

### 检查构建输出

在 Cloudflare Dashboard 的构建日志中，确认：
- ✅ 构建成功完成
- ✅ 输出目录是 `dist`
- ✅ 没有资源相关的错误

### 检查 dist 目录结构

如果可能，检查构建后的 `dist` 目录应该包含：
```
dist/
├── _nuxt/
│   ├── assets/
│   │   ├── js/
│   │   ├── css/
│   │   └── images/
│   └── ...
├── _worker.js/
└── ...
```

### 可能的额外修复

如果上述配置仍然不行，可能需要：

1. **检查 Cloudflare Pages 的路由配置**
   - 在 Cloudflare Dashboard → Pages → 你的项目 → Settings → Functions
   - 确认没有自定义路由规则阻止 `/_nuxt/` 路径

2. **尝试使用不同的 buildAssetsDir**
   如果 `/_nuxt/` 路径仍然有问题，可以尝试：
   ```typescript
   app: {
     buildAssetsDir: '/', // 将资源放在根目录
   }
   ```
   但这需要重新构建和测试。

3. **检查 Nitro 版本**
   确保使用的是最新版本的 Nitro，旧版本可能有路径问题。
