## 🚀 使用方法

### 前端：申请友链

1. 访问 `/friends` 页面
2. 点击"申请友链"按钮
3. 填写表单：
   - 网站名称（必填）
   - 网站链接（必填）
   - 网站描述（可选）
   - 网站图标（可选）
4. 完成 Turnstile 验证
5. 点击"提交申请"

### 后台：管理友链

1. 访问 `/admin` 页面
2. 登录后，点击"友链管理"标签
3. **待审核申请**：
   - 查看所有申请
   - 点击"批准"将友链添加到列表
   - 点击"拒绝"标记为已拒绝
   - 点击"删除"删除申请
4. **已批准友链**：
   - 查看所有已批准的友链
   - 点击"删除"移除友链

## 📊 数据存储结构

### KV 存储结构

```
friend-requests:list          → ["1", "2", "3"]  // 申请 ID 列表
friend-request:1              → { id, name, url, description, avatar, status, createdAt, ip }
friend-request:2              → { id, name, url, description, avatar, status, createdAt, ip }
...

friends:list                  → ["1", "3"]  // 已批准友链 ID 列表
friend:1                      → { id, name, url, description, avatar, date }
friend:3                      → { id, name, url, description, avatar, date }
```

### 申请状态

- `pending` - 待审核
- `approved` - 已批准（会添加到友链列表）
- `rejected` - 已拒绝

## 🔒 安全特性

1. **Turnstile 验证** - 防止机器人提交申请
2. **管理员认证** - 只有管理员可以批准/拒绝申请
3. **IP 记录** - 记录申请者的 IP 地址

## ⚠️ 注意事项

1. **Turnstile 配置** - 如果配置了 Turnstile，申请时需要完成验证
2. **数据迁移** - 现有的 `data/friends.json` 文件不会被使用，需要手动迁移或重新申请
3. **头像路径** - 头像 URL 可以是完整 URL 或相对路径（如 `/images/avatar.webp`）

## 🎯 工作流程

1. **用户申请**：
   - 填写表单 → 完成验证 → 提交申请
   - 申请保存到 KV，状态为 `pending`

2. **管理员审核**：
   - 查看申请列表
   - 批准 → 友链添加到列表，状态变为 `approved`
   - 拒绝 → 状态变为 `rejected`
   - 删除 → 从 KV 中删除

3. **前端显示**：
   - 只显示状态为 `approved` 的友链
   - 自动从 API 加载最新数据
