# 🔧 R2 自定义域名配置说明

## ✅ 已修复

我已经更新了代码，现在上传图片后会返回完整的 URL（使用你的自定义域名 `photo.lcjlq.com`）。

## 📝 配置步骤

### 方法 1：使用环境变量（推荐）

在 Cloudflare Pages 中设置环境变量：

1. 登录 Cloudflare Dashboard
2. 进入你的 Pages 项目
3. 点击 **Settings** → **Environment variables**
4. 添加环境变量：
   - **Variable name**：`R2_PUBLIC_URL`
   - **Value**：`https://photo.lcjlq.com`
   - **Environment**：Production（或所有环境）

### 方法 2：修改代码中的默认值

如果不想使用环境变量，可以直接修改 `nuxt.config.ts`：

```typescript
r2PublicUrl: process.env.R2_PUBLIC_URL || 'https://photo.lcjlq.com',
```

## 🔄 重新部署

配置完成后，重新部署项目：

```bash
git add .
git commit -m "修复 R2 图片 URL - 使用自定义域名"
git push
```

## ✅ 验证

部署完成后：

1. 访问 `/admin` 页面
2. 上传一张新图片
3. 检查返回的 URL 格式：
   - ✅ 应该是：`https://photo.lcjlq.com/covers/xxx.png`
   - ❌ 不应该是：`/r2/covers/xxx.png` 或 `/api/r2/covers/xxx.png`

## 🎯 URL 格式说明

- **如果配置了 `R2_PUBLIC_URL`**：返回完整 URL `https://photo.lcjlq.com/covers/xxx.png`
- **如果没有配置**：返回代理路由 `/api/r2/covers/xxx.png`（通过 `/api/r2/[...path]` 代理）

## ⚠️ 注意事项

1. **确保自定义域名已正确配置**
   - R2 bucket → Settings → Public access
   - 确认自定义域名 `photo.lcjlq.com` 已配置并生效

2. **DNS 配置**
   - 确保 `photo.lcjlq.com` 的 DNS 记录指向 Cloudflare
   - 确保 SSL/TLS 证书已配置

3. **CORS 配置（如果需要）**
   - 如果图片需要在其他域名显示，可能需要配置 CORS

## 🔍 如果图片仍然不显示

1. **检查 URL 是否正确**
   - 在浏览器中直接访问图片 URL
   - 例如：`https://photo.lcjlq.com/covers/1765053996815-d27450a.png`

2. **检查浏览器控制台**
   - 查看是否有 CORS 错误
   - 查看图片加载错误信息

3. **检查 R2 bucket 设置**
   - 确认公共访问已启用
   - 确认自定义域名已配置
