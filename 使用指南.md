# 🎉 博客后台管理系统 - 快速使用指南

## ✅ 改造完成！

你的博客已经成功改造为支持后台管理的版本。现在你可以通过 Web 界面管理博客文章，无需手动创建 markdown 文件！

## 🚀 快速开始（3 步）

### 第 1 步：配置 Cloudflare KV

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入你的 Workers & Pages 项目
3. 点击左侧菜单 "KV" → "Create a namespace"
4. 命名空间名称：`BLOG_KV`
5. 创建后，复制命名空间的 **ID**

### 第 2 步：更新配置文件

编辑 `wrangler.jsonc`，填入你的 KV 命名空间 ID：

```jsonc
{
  "kv_namespaces": [
    {
      "binding": "BLOG_KV",
      "id": "你的-kv-命名空间-id",  // 👈 替换这里
      "preview_id": "你的预览-id"    // 👈 可选，用于本地开发
    }
  ]
}
```

### 第 3 步：设置管理员密码

在 Cloudflare Dashboard 中：

1. 进入你的项目 → "Settings" → "Variables"
2. 添加环境变量：
   - **变量名**：`ADMIN_PASSWORD`
   - **变量值**：设置一个强密码（例如：`MySecurePassword123!`）

## 📝 使用后台管理

### 访问后台

部署后访问：`https://你的域名/admin`

### 登录

使用你设置的 `ADMIN_PASSWORD` 登录。

### 功能

- ✅ **创建文章**：点击 "新建文章"，填写标题和 Markdown 内容
- ✅ **编辑文章**：在文章列表中点击 "编辑"
- ✅ **删除文章**：在文章列表中点击 "删除"

## 🔄 迁移现有文章（可选）

如果你有现有的 `posts/*.md` 文件，有两种方式迁移：

### 方式 1：手动迁移（推荐）

直接在后台管理界面创建文章，复制现有 markdown 文件的内容。

### 方式 2：使用迁移 API

在本地环境运行：

```bash
# 1. 先登录获取 token
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"password":"你的密码"}'

# 2. 使用返回的 token 运行迁移
curl -X POST http://localhost:3000/api/migrate \
  -H "Authorization: Bearer 你的token"
```

## 📦 部署

```bash
# 构建项目
npm run build

# 部署到 Cloudflare Pages
npx wrangler pages deploy dist
```

## ⚠️ 重要提示

1. **首次使用**：KV 是空的，需要先在后台创建文章
2. **兼容性**：如果 API 不可用，系统会自动回退到从本地文件读取（兼容模式）
3. **安全**：确保使用强密码，并启用 HTTPS

## 🎯 技术说明

- **存储**：使用 Cloudflare KV 存储文章数据
- **API**：RESTful API 提供增删改查功能
- **认证**：简单的密码认证（Bearer Token）
- **兼容**：支持从 API 或本地文件系统读取文章

## ❓ 遇到问题？

1. 检查 KV 命名空间是否正确配置
2. 检查环境变量 `ADMIN_PASSWORD` 是否设置
3. 查看浏览器控制台的错误信息
4. 查看 Cloudflare Dashboard 的日志
