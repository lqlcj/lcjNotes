# 🔍 图片上传功能调试指南

## ✅ 已修复的问题

我已经添加了缺失的上传函数：
- `triggerFileInput()` - 触发文件选择器
- `handleFileSelect()` - 处理文件选择
- `uploadImage()` - 上传图片到 R2
- `clearCoverPreview()` - 清除预览

## 🔧 排查步骤

### 步骤 1：检查浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到 **Console** 标签
3. 点击"上传图片"按钮
4. 查看是否有 JavaScript 错误

**常见错误：**
- `triggerFileInput is not defined` - 说明函数未定义（已修复）
- `Cannot read property 'click' of null` - 说明 fileInput ref 未正确绑定

### 步骤 2：检查 R2 配置

1. 登录 Cloudflare Dashboard
2. 进入你的 Pages 项目
3. 点击 **Settings** → **Functions**
4. 检查 **R2 bucket bindings**：
   - ✅ Variable name 必须是 `BLOG_R2`（大小写敏感）
   - ✅ 必须选择了正确的 R2 bucket

### 步骤 3：测试上传 API

在浏览器控制台中运行：

```javascript
// 测试上传 API（需要先登录）
const token = localStorage.getItem('admin_token');
const formData = new FormData();
// 这里需要选择一个文件
// formData.append('file', file);

fetch('/api/upload/image', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
})
.then(r => r.json())
.then(console.log)
.catch(console.error);
```

### 步骤 4：检查网络请求

1. 打开浏览器开发者工具（F12）
2. 切换到 **Network** 标签
3. 点击"上传图片"按钮
4. 查看是否有 `/api/upload/image` 请求
5. 如果有请求，查看响应状态码和错误信息

**常见错误：**
- `401 Unauthorized` - 未登录或 token 无效
- `500 Internal Server Error` - 服务器错误，可能是 R2 binding 未配置
- `R2 storage not available` - R2 binding 未正确配置

## 🐛 常见问题

### Q1: 点击按钮没反应

**可能原因：**
1. JavaScript 函数未定义（已修复）
2. 按钮被禁用（检查 `uploading` 状态）
3. 事件监听器未绑定

**解决方法：**
1. 刷新页面，确保使用最新代码
2. 检查浏览器控制台是否有错误
3. 确认已重新部署项目

### Q2: 选择文件后没有反应

**可能原因：**
1. 文件类型不支持
2. 文件大小超过限制（10MB）
3. `handleFileSelect` 函数出错

**解决方法：**
1. 检查文件类型（仅支持 JPEG, PNG, WebP, GIF）
2. 检查文件大小
3. 查看浏览器控制台错误信息

### Q3: 上传失败，提示 "R2 storage not available"

**原因：** R2 binding 未正确配置

**解决方法：**
1. 检查 Cloudflare Pages 的 R2 binding 配置
2. 确认 Variable name 是 `BLOG_R2`（大小写敏感）
3. 确认选择了正确的 R2 bucket
4. 重新部署项目

### Q4: 上传失败，提示 "未授权访问"

**原因：** 未登录或 token 无效

**解决方法：**
1. 重新登录后台管理
2. 检查 `localStorage.getItem('admin_token')` 是否有值
3. 确认管理员密码正确
