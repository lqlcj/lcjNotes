# 🔗 友链申请功能使用指南

## ✅ 功能概述

友链申请功能已完全迁移到 KV 存储，支持：
- ✅ 用户提交友链申请（带 Turnstile 验证）
- ✅ 管理员在后台审核申请
- ✅ 批准后自动显示在前端
- ✅ 所有数据存储在 Cloudflare KV 中

## 🎯 工作流程

### 1. 用户申请友链

1. 访问 `/friends` 页面
2. 点击"申请友链"按钮
3. 填写表单：
   - **网站名称**（必填）
   - **网站链接**（必填，需要包含 http:// 或 https://）
   - **网站描述**（可选）
   - **网站图标**（可选，URL 格式）
4. 完成 Turnstile 人机验证
5. 点击"提交申请"

### 2. 管理员审核

1. 访问 `/admin` 页面
2. 登录后，点击"友链管理"标签
3. 查看"待审核申请"列表
4. 操作选项：
   - **批准** - 将友链添加到已批准列表，前端会显示
   - **拒绝** - 标记为已拒绝（不会显示在前端）
   - **删除** - 删除申请记录

### 3. 前端显示

- 只显示状态为 `approved` 的友链
- 自动从 API 加载最新数据
- 按添加日期倒序排列

## 📊 数据存储

### KV 存储结构

```
friend-requests:list          → ["1", "2", "3"]  // 所有申请 ID
friend-request:1               → { id, name, url, description, avatar, status: "pending", createdAt, ip }
friend-request:2               → { id, name, url, description, avatar, status: "approved", createdAt, ip }
...

friends:list                   → ["2", "5"]  // 已批准友链 ID
friend:2                        → { id, name, url, description, avatar, date }
friend:5                        → { id, name, url, description, avatar, date }
```

### 申请状态

- `pending` - 待审核（默认）
- `approved` - 已批准（会出现在前端）
- `rejected` - 已拒绝（不会出现在前端）

## 🔒 安全特性

1. **Turnstile 验证** - 防止机器人提交申请
2. **管理员认证** - 只有管理员可以审核申请
3. **IP 记录** - 记录申请者的 IP 地址

## 📝 API 接口

### 公开接口

- `POST /api/friends/requests` - 提交友链申请（需要 Turnstile 验证）
- `GET /api/friends` - 获取已批准友链列表

### 管理员接口（需要认证）

- `GET /api/friends/requests` - 获取所有申请列表
- `PUT /api/friends/requests/:id` - 更新申请状态（批准/拒绝）
- `DELETE /api/friends/requests/:id` - 删除申请
- `DELETE /api/friends/:id` - 删除已批准友链

## ⚠️ 注意事项

1. **Turnstile 配置** - 如果配置了 Turnstile，申请时必须完成验证
2. **数据迁移** - 现有的 `data/friends.json` 文件不会被使用
   - 如果需要迁移现有友链，可以在后台手动添加
   - 或者批准新的申请
3. **头像路径** - 支持：
   - 完整 URL：`https://example.com/avatar.jpg`
   - 相对路径：`/images/avatar.webp`
   - 如果不提供，会使用默认头像
